# 📄 Документация проекта: «Cyber-Socratic Arena» (Архитектор Смыслов)

---

## 1. Описание и цель проекта

Приложение создано для решения проблемы бездумного копирования ответов от ИИ (Сценарий А) и перевода учеников к осознанному обучению с помощью нейросетей (Сценарий В).

**Главная цель:** Не просто дать ученику доступ к ИИ, а научить его учиться с помощью ИИ грамотно, осознанно и эффективно, развивая навыки промпт-инжиниринга и критического мышления.

---

## 2. Основной функционал и игровые механики

- **Барьер «Что я знаю»:** Перед началом работы ученик не вводит обычный промпт, а кратко пишет всё, что уже знает по теме (даже если немного). Только после этого активируется ИИ.

- **Сократовский метод обучения:** ИИ не выдаёт ответ сразу. Он работает как наставник, который ведёт ученика через наводящие вопросы и подсказки к самостоятельному пониманию.

- **Финальная рефлексия:** В конце ученик своими словами пишет, что он понял. ИИ даёт доброжелательный фидбэк и помогает закрепить знания.

- **Тренировка промптов:** Приложение помогает ученику улучшать качество запросов, объясняя, как правильно и точно формулировать промпты.

- **Мини-игра с карточками:** Соединение пар (термин-определение, формула-значение) для лучшего запоминания.

- **Мониторинг для учителя:** Система отслеживает прогресс, слабые места и выученный материал, передавая данные учителю. Доступен рейтинг класса.

## 3. Архитектура приложения (Для команды разработки)

Система делится на 3 основных слоя:

### 3.1. Frontend (Пользовательский интерфейс)

- **Технологии:** Web-приложение (React + TypeScript). Роутинг через React Router v6. Стейт — через Context API или Zustand.
- **Анимации:** CSS-анимации и Framer Motion для плавного появления текста, светящихся элементов и градиентов.
- **Сборка:** Vite. Деплой через Vercel или Netlify.

### 3.2. Backend & AI Proxy Layer (Серверная часть и логика ИИ)

- **Умный Прокси-слой:** Ученик не общается с API нейросети (Claude/OpenAI) напрямую. Backend перехватывает запрос ученика и «оборачивает» его в жёсткий системный промпт:
  ```
  System Instruction: "Ты — наставник. Никогда не давай готовый ответ.
  Отвечай наводящим вопросом, веди ученика к самостоятельному пониманию."
  ```
- **Аналитика:** Сбор метрик (количество попыток, качество промптов, пройденные уровни) для записи в базу данных.
- **API маршруты:**
  - `POST /api/chat` — отправить сообщение ИИ (прокси с системным промптом)
  - `GET /api/progress/:userId` — получить прогресс ученика
  - `GET /api/teacher/dashboard` — получить данные дашборда учителя (только для роли teacher)

### 3.3. Teacher Module (Панель учителя)

- **Дашборд:** Визуализация данных из базы. Учитель видит «пульс класса», рейтинг учеников, график прогресса (кто думает сам, кто зависает) и проблемные темы.
- **Доступ:** Только аккаунты с ролью `teacher` имеют доступ к этому разделу. При попытке ученика зайти — редирект на `/student`.

- Роль `student` → редирект на `/student`

### 4.4. Защита маршрутов

Создать компонент `<ProtectedRoute role="teacher">` / `<ProtectedRoute role="student">`. Логика:

```js
// src/components/ProtectedRoute.jsx
const ProtectedRoute = ({ role, children }) => {
  const user = JSON.parse(localStorage.getItem("currentUser"));
  if (!user) return <Navigate to="/login" />;
  if (user.role !== role) return <Navigate to={`/${user.role}`} />;
  return children;
};
```

Роутинг приложения:
```
/login              → LoginPage (публичный)
/student            → StudentPage (только role: student)
/student/chat       → ChatPage (только role: student)
/teacher            → TeacherDashboard (только role: teacher)
```

## 5. UI/UX Дизайн-система (Strict Guidelines)

**Общий стиль:** Геймифицированный, современный интерфейс, снижающий нагрузку на глаза (Dark Mode). Никаких острых углов, формы гладкие и скругленные. Текст максимально чистый (шрифты `Inter` или `Roboto`). Иконки в стиле Neon Duo-tone.

**Цветовая палитра:**

| Роль | Цвет |
|---|---|
| Фон (Base) | Глубокий Космический Синий `#0D1117` или Тёмный Грифельный `#1A1D2E` |
| Акценты | Вибрирующие градиенты (`#6C63FF` → `#00D4FF`) |
| Успех / Действие | Неоновый Бирюзовый / Зелёный `#00E5A0` |
| Награды / Магия | Электрический Фиолетовый `#A855F7` / Розовый `#EC4899` |
| Ошибки (Growth Mindset) | Мягкий Оранжевый `#F97316` (не агрессивный красный) |

## 6. Структура экранов (Figma / Web-Макеты)

### Экран 0: Авторизация (`/login`)

- Центрированная карточка на тёмном фоне с логотипом проекта сверху.
- Два поля: Email и Пароль. Кнопка «Войти» с градиентом.
- Под формой — подсказка с демо-аккаунтами:
  ```
  Ученик:  student@demo.com / student123
  Учитель: teacher@demo.com / teacher123
  ```
- После входа — автоматический редирект по роли.

### Экран 1: Чат «Сократовского Разогрева» (`/student/chat`)

- **Вид:** Минималистичный мессенджер с премиальным дизайном.
- **Аватар ИИ:** Анимированный «Дух Знаний» / неоновый шар. Пульсирует во время «раздумий», меняет цвет в зависимости от тона диалога.
- **Барьер:** Перед первым сообщением — модальное окно или блок «Что ты уже знаешь по этой теме?». Поле чата разблокируется только после заполнения.
- **Ввод текста:** Поле выделено ярким градиентом, текст появляется плавно (анимация `fadeIn`).

### Экран 2: Карта «Прогрессивного Квеста» (`/student`)

- **Вид:** Визуализированная карта созвездий или игровой мир.
- **Механика:** 15 точек-уровней. Доступные уровни ярко светятся. Закрытые — скрыты под «туманом войны» или перекрыты неоновыми цепями.
- **Прогресс:** Полоска XP (опыта) на карте, заполняется с анимацией при решении задач.
- **Клик по уровню** → переход на `/student/chat` с предзагруженной темой урока.

### Экран 3: «Холст Мышления» (Интерактивная зона)

- Рабочая область, где ИИ выдаёт не сплошной текст, а блоки информации («карточки идей»).
- Ученик вручную соединяет эти блоки, выстраивая логическую структуру ответа.
- Реализуется через drag-and-drop (библиотека `@dnd-kit/core`).

### Экран 4: Дашборд учителя (`/teacher`)

- **Вид:** Аналитическая панель.
- **Компоненты:**
  - Карточки-метрики вверху: «Всего учеников», «Активны сегодня», «Средний XP класса».
  - Таблица учеников: имя, уровень, XP, `promptQuality`, `thinkingIndependence`, последняя активность.
  - Прогресс-бары для каждого ученика (визуализируют `thinkingIndependence`).
  - Блок «Проблемные темы класса»: агрегирует `weakTopics` всех учеников и показывает топ-3.
- **Данные:** Берутся из `MOCK_PROGRESS`. Реальных API-запросов не требуется.
